ОТБОРОЧНОЕ ЗАДАНИЕ НАПРАВЛЕНИЯ «ФУЛСТЭК»
Кейс: «Монитор подписок»
ПРЕАМБУЛА:
В условиях роста популярности моделей монетизации на основе регулярных платежей,
управление множеством персональных и корпоративных подписок становится сложной
задачей. Пользователи сталкиваются с трудностями отслеживания сроков списаний, анализа
целесообразности расходов и оперативного управления статусами подписок в различных
сервисах. Создание единой платформы для контроля всех активных подписок,
интегрированной с популярными в России платежными решениями и почтовыми сервисами,
позволит пользователям оптимизировать регулярные расходы и избегать нежелательных
списаний. Приложение будет агрегировать данные о подписках, предоставлять детальную
аналитику и давать инструменты для управления ими из одного интерфейса.
ЗАДАЧА:
Разработать кроссплатформенное решение, состоящее из мобильного приложения
(iOS/Android), веб-приложения и общего сервера. Функционал должен включать:
• Централизованное отображение всех активных подписок пользователя (стриминги,
программное обеспечение, сервисы доставки, облачные хранилища и т.д.) в единой панели
управления.
• Автоматический сбор данных о подписках: интеграция с платежными системами
(Сбербанк Online, Тинькофф, ЮMoney и другие) и/или безопасный парсинг входящей почты
(Mail.ru, Yandex, Gmail) на наличие чеков и уведомлений о списаниях (строго с явного
согласия пользователя).
• Визуализацию аналитики по затратам с группировкой по категориям, сервисам и
периодам.
• Функцию анализа использования подписок на основе данных, предоставленных
пользователем (например, отметки о просмотре контента, использование софта).
• Формирование прогноза трат на подписки на год вперед.
• Настройку и отправку умных уведомлений: предупреждение о предстоящем списании,
повышении цены, неактивности в сервисе.
• Возможность поиска и рекомендаций альтернативных, более выгодных сервисов
внутри той же категории.
• Функцию быстрой отмены или приостановки подписки с перенаправлением в
соответствующий сервис или формированием шаблона письма для поддержки.
ЧТО МОЖНО ИСПОЛЬЗОВАТЬ:
⎯ Android studio / VS-code или любая другая среда разработки;
⎯ Языки программирования: Java/ Kotlin/ React Native / php / node-js / python/ любой
другой;
⎯ ПО для создания баз данных (при необходимости);
⎯ нейросети.
