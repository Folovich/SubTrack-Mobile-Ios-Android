# SubTrack Mobile

Мобильное приложение SubTrack для iOS и Android на базе Expo и React Native.

## Что это за проект

`SubTrack Mobile` не является отдельным backend-сервисом.
Это mobile-клиент и дополнение к backend SubTrack.

Приложение работает только вместе с запущенным backend:

- авторизация
- подписки
- аналитика
- интеграции
- импорт

всё это приходит с backend API.

## Возможности

- регистрация и вход
- просмотр дашборда
- управление подписками
- ближайшие списания
- уведомления
- аналитика расходов
- импорт данных
- интеграции
- настройки приложения

## Настройка backend

Приложение использует переменную окружения `EXPO_PUBLIC_API_URL`.

1. Откройте файл `.env` в корне проекта.
2. Укажите в нём адрес backend в переменной `EXPO_PUBLIC_API_URL`.

Примеры:

```env
# Android Emulator
EXPO_PUBLIC_API_URL=http://10.0.2.2:8080
```

```env
# iOS Simulator
EXPO_PUBLIC_API_URL=http://localhost:8080
```

```env
# Expo Go на физическом устройстве в той же Wi-Fi сети, что и backend
EXPO_PUBLIC_API_URL=http://192.168.1.100:8080
```

## Какой адрес использовать

- `http://localhost:8080` или `http://127.0.0.1:8080`
  Используйте только если backend запущен на этом же компьютере и приложение открыто в `iOS Simulator` или в браузере.

- `http://10.0.2.2:8080`
  Используйте для `Android Emulator`, если backend запущен на вашем компьютере.

- `http://<ВАШ_IP>:8080`
  Используйте на физическом телефоне через `Expo Go`. Телефон и компьютер с backend должны быть в одной локальной сети.

## Важно про импорт почты

Импорт почты и Gmail mailbox sync зависят от backend и OAuth-сценария.

- На физическом устройстве `localhost` для почты не подойдёт.
- Если mobile открыт через `Expo Go`, а backend указан как `localhost`, телефон будет искать backend на самом себе.
- В таком режиме почта не будет нормально парситься, OAuth redirect и mailbox sync работать не будут.

Для почтового импорта используйте backend, доступный по локальному IP или домену, например:

```env
EXPO_PUBLIC_API_URL=http://192.168.1.100:8080
```

## Как узнать свой IP

Windows:

```bash
ipconfig
```

Найдите IPv4-адрес активного адаптера, например `192.168.1.100`, и подставьте его в `.env`.

## Запуск приложения

1. Убедитесь, что backend уже запущен и доступен по адресу из `.env`.
2. Запустите Expo:

```bash
npm start
```

3. Выберите способ запуска:

- нажмите `a`, чтобы открыть Android Emulator
- нажмите `i`, чтобы открыть iOS Simulator
- отсканируйте QR-код через `Expo Go` на телефоне

## Важно

- Если приложение запускается на телефоне, `localhost` работать не будет.
- Если backend запущен в Docker, убедитесь, что порт `8080` проброшен наружу.
- Если телефон не подключается к backend по IP, проверьте firewall и что оба устройства находятся в одной сети.
- Mobile-приложение является клиентским модулем для backend SubTrack, поэтому без backend большая часть функций работать не будет.
